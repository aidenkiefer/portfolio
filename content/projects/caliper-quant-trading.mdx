## Overview

Caliper is a modular quantitative ML trading platform for US equities that covers the full lifecycle: market data ingestion, feature engineering, backtesting, risk-controlled execution via a broker adapter, ML training/inference, drift monitoring, and a Next.js dashboard (“Model Observatory”).

The design goal was safety and correctness: build guardrails and observability before trusting model-driven signals.

## Problem & Context

Most retail algorithmic trading tools are either opaque or unsafe: they don’t model risk rigorously, and ML signals are hard to trust without explainability and drift monitoring. Caliper was built to make the system inspectable and conservative by default.

## Constraints

- Paper trading by default; promote to live only behind explicit configuration.
- Secrets must remain out of source control (.env examples only).
- All orders must route through a central RiskManager (no bypass).

## Approach & Design Decisions

- **Service-oriented monorepo**: Python services + shared schema packages + a Next.js dashboard in one repo for atomic changes.
- **TimescaleDB** for time-series price bars with hypertables.
- **BFF pattern**: dashboard calls FastAPI, with rewrites/proxying to avoid leaking backend URLs.
- **ML safety before ML performance**: confidence gating, drift checks, explainability, and HITL workflow were implemented before the first integrated model.

## Implementation Highlights

- Market data ingestion into TimescaleDB.
- Feature pipeline (dozens of technical + derived features).
- Backtesting engine with performance metrics and reporting.
- Walk-forward optimization over strategy parameters.
- Risk controls: drawdown limits, position caps, notional caps, and per-strategy constraints.
- Execution OMS with idempotency via `client_order_id` patterns.
- Dashboard surfaces health, model details, and recommendation approvals.
